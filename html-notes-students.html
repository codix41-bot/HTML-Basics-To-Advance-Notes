Chapter 01

What is HTML?

HTML is the skeleton of every webpage you visit. Before learning any framework, you must master HTML.

Definition

HTML (HyperText Markup Language) is the standard language used to create and structure content on the web. It uses tags to tell the browser what each piece of content is — a heading, a paragraph, a link, an image, etc.

Key Terms to Know

TermMeaning
TagA keyword inside angle brackets, e.g. <p>
ElementAn opening tag + content + closing tag: <p>Hello</p>
AttributeExtra info inside an opening tag: <img src="pic.jpg">
ValueThe data assigned to an attribute: "pic.jpg"
NestingPlacing elements inside other elements
DOMDocument Object Model — the browser's tree of all elements
💡 TIP
HTML is NOT a programming language — it has no logic, no variables, no loops. It is a markup language. Programming comes with JavaScript.
Chapter 02

Basic HTML Structure

Every HTML file must follow a basic skeleton. This is the minimum a browser needs to render a valid page.

HTML 
<!-- This is a complete HTML page -->
<!DOCTYPE html>                  <!-- Tells browser: this is HTML5 -->
<html lang="en">                    <!-- Root element -->

  <head>                           <!-- Invisible info for browser -->
    <meta charset="UTF-8">          <!-- Support all characters -->
    <meta name="viewport" content="width=device-width">
    <title>My First Page</title>    <!-- Tab title -->
  </head>

  <body>                           <!-- Visible content goes here -->
    <h1>Hello, World!</h1>
    <p>This is my first webpage.</p>
  </body>

</html>

What Each Part Does

TagPurpose
<!DOCTYPE html>Declaration — tells browser to use HTML5 mode
<html>Root of the entire document
<head>Metadata — not visible on page (title, CSS links, scripts)
<body>Everything the user sees goes here
<title>Text shown in browser tab
<meta charset>Sets character encoding (always use UTF-8)
⚠ IMPORTANT
Always include <!DOCTYPE html> as the very first line. Without it, browsers enter "quirks mode" and your page may look broken.
Chapter 03

Text Elements

Text is the core of most webpages. HTML has many tags specifically for organizing and formatting text.

Headings — h1 to h6

Definition

Headings define the hierarchy of your content. <h1> is the most important (biggest), <h6> is the least important (smallest). Use only ONE <h1> per page.

HTML
<h1>Main Title (Biggest)</h1>
<h2>Section Heading</h2>
<h3>Sub-section</h3>
<h4>Sub-sub-section</h4>
<h5>Minor Heading</h5>
<h6>Smallest Heading</h6>

Paragraph & Line Break

HTML
<p>This is a paragraph. Browsers automatically
add space before and after paragraphs.</p>

<p>First line of text.<br>This starts on a new line (no gap).</p>

<hr>  <!-- Horizontal rule: a dividing line -->

Text Formatting Tags

<strong>Bold + important
<em>Italic + emphasis
<u>Underline
<s>Strikethrough
<mark>Highlighted text
<code>Inline code
<pre>Preformatted text
<blockquote>Long quotation
<abbr>Abbreviation tooltip
<sup>Superscript: x²
<sub>Subscript: H₂O
<small>Smaller text
HTML — Text Formatting Examples
<p>This is <strong>bold and important</strong> text.</p>
<p>This is <em>italicized</em> for emphasis.</p>
<p>Water formula: H<sub>2</sub>O</p>

Chapter 04

Links & Images

Links connect pages. Images add visuals. Together they make the web rich and navigable.

Hyperlinks — <a> tag

Definition

The <a> (anchor) tag creates clickable links. The href attribute specifies the destination URL.

HTML
<!-- External link (opens in new tab) -->
<a href="https://google.com" target="_blank">Visit Google</a>

<!-- Internal link (same website) -->
<a href="about.html">About Us</a>

<!-- Jump to section on same page -->
<a href="#contact">Go to Contact</a>

<!-- Email link -->
<a href="mailto:hello@email.com">Email Me</a>

<!-- Phone call link -->
<a href="tel:+911234567890">Call Us</a>
AttributeValueEffect
hrefURL / pathWhere the link goes
target_blankOpens in new tab
target_selfOpens in same tab (default)
relnoopenerSecurity for _blank links
downloadfilenameForces file download

Images — <img> tag

Definition

<img> is a self-closing tag (no closing tag needed). The src attribute is the image path. The alt attribute describes the image for screen readers and when image fails to load.

HTML
<!-- Basic image -->
<img src="photo.jpg" alt="A beautiful sunset">

<!-- With width and height -->
<img src="logo.png" alt="Company Logo"
     width="200" height="100">

<!-- Image from the internet -->
<img src="https://example.com/pic.jpg"
     alt="Description"
     loading="lazy">   <!-- lazy = loads only when visible -->

<!-- Clickable image (link + image) -->
<a href="https://google.com">
  <img src="banner.jpg" alt="Click to visit">
</a>
Chapter 05

Lists

Lists organize information into digestible, scannable items.

Three Types of Lists

HTML
<!-- 1. Unordered list (bullets) -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- 2. Ordered list (numbers) -->
<ol>
  <li>Open the editor</li>
  <li>Write your HTML</li>
  <li>Save and open in browser</li>
</ol>

<!-- 3. Description list (term + definition) -->
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
</dl>

<!-- Nested list -->
<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </li>
  <li>Backend</li>
</ul>
Chapter 06

Tables

Tables display data in rows and columns. Use them for data only — not for page layout.

Definition

A table has three sections: <thead> (header row), <tbody> (data rows), and <tfoot> (footer row). Each row is <tr>, header cells are <th>, and data cells are <td>.

HTML — Complete Table
<table border="1">

  <thead>
    <tr>
      <th>Name</th>
      <th>Age</th>
      <th>City</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>Rahul</td>
      <td>21</td>
      <td>Delhi</td>
    </tr>
    <tr>
      <td>Priya</td>
      <td>23</td>
      <td>Mumbai</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <td colspan="3">Total: 2 students</td>
    </tr>
  </tfoot>

</table>
AttributeUsed OnEffect
colspantd / thMerge across multiple columns
rowspantd / thMerge across multiple rows
scopethAccessibility: row or col
Chapter 07

Forms & Inputs

Forms collect user data — registration, login, search, contact. This is one of the most important chapters.

HTML — Complete Form Example
<form action="/submit" method="POST">

  <!-- Text input -->
  <label for="name">Full Name:</label>
  <input type="text" id="name" name="name"
         placeholder="Enter your name" required>

  <!-- Email input -->
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>

  <!-- Password -->
  <input type="password" name="pass" minlength="8">

  <!-- Number with range -->
  <input type="number" min="1" max="100">

  <!-- Radio buttons -->
  <input type="radio" name="gender" value="male"> Male
  <input type="radio" name="gender" value="female"> Female

  <!-- Checkboxes -->
  <input type="checkbox" name="html" checked> HTML
  <input type="checkbox" name="css"> CSS

  <!-- Dropdown -->
  <select name="city">
    <option value="">Select city</option>
    <option value="delhi">Delhi</option>
    <option value="mumbai">Mumbai</option>
  </select>

  <!-- Textarea -->
  <textarea name="msg" rows="5" cols="40"></textarea>

  <!-- File upload -->
  <input type="file" accept=".jpg,.png">

  <!-- Date picker -->
  <input type="date" name="dob">

  <!-- Submit button -->
  <button type="submit">Submit Form</button>

</form>

Chapter 08

Semantic HTML

Semantic elements clearly describe their meaning to the browser and developers. This is the most important skill for writing professional HTML.

Definition

Semantic HTML means using elements that describe what they contain, not how they look. <div> and <span> are non-semantic (generic containers). <header>, <article>, <nav> are semantic — they tell the browser exactly what role that section plays.

HTML — Page Layout with Semantic Tags
<body>

  <header>           <!-- Site header / logo / top nav -->
    <nav>             <!-- Navigation links -->
      <a href="/">Home</a>
    </nav>
  </header>

  <main>            <!-- Main unique content (only 1 per page) -->

    <section>       <!-- A themed group of content -->
      <h2>Latest News</h2>

      <article>     <!-- Standalone piece of content -->
        <h3>News Title</h3>
        <p>Article content...</p>
      </article>

    </section>

    <aside>           <!-- Sidebar / related content -->
      <p>Related links...</p>
    </aside>

  </main>

  <footer>          <!-- Copyright, links at bottom -->
    <p>&copy; 2025 My Website</p>
  </footer>

</body>
Semantic TagPurpose
<header>Top of page or section — logo, nav
<nav>Navigation menu links
<main>Core unique content of the page (only one!)
<section>A thematic grouping with a heading
<article>Self-contained content (blog post, news, comment)
<aside>Sidebar or tangentially related content
<footer>Bottom of page or section
<figure>Self-contained media (image, chart, code)
<figcaption>Caption for a figure
<time>Machine-readable date or time
💡 WHY IT MATTERS
Semantic HTML improves: (1) SEO — Google understands your content better, (2) Accessibility — screen readers navigate by landmarks like <main> and <nav>, (3) Maintainability — other developers understand your code faster.
Chapter 09

HTML5 Media

HTML5 introduced native audio and video — no plugins required.

HTML — Audio & Video
<!-- VIDEO -->
<video
  src="movie.mp4"
  width="640"
  height="360"
  controls         <!-- Show play/pause controls -->
  autoplay         <!-- Play automatically -->
  muted            <!-- Required for autoplay in browsers -->
  loop             <!-- Repeat forever -->
  poster="thumb.jpg">
  Your browser does not support video.
</video>

<!-- Multiple formats for compatibility -->
<video controls>
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.webm" type="video/webm">
</video>

<!-- AUDIO -->
<audio controls>
  <source src="song.mp3" type="audio/mpeg">
  <source src="song.ogg" type="audio/ogg">
</audio>

<!-- IFRAME: embed another page -->
<iframe
  src="https://www.youtube.com/embed/VIDEO_ID"
  width="560" height="315"
  allowfullscreen>
</iframe>
Chapter 10 — Advanced

Meta Tags & SEO

Meta tags live in <head> and tell search engines and social media platforms about your page.

HTML — Complete Head for SEO
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- SEO basics -->
  <title>Page Title — Under 60 chars</title>
  <meta name="description"
        content="A 150-160 char description shown in search results.">
  <meta name="keywords" content="html, css, web development">
  <meta name="author" content="Your Name">
  <meta name="robots" content="index, follow">

  <!-- Open Graph: for Facebook / WhatsApp sharing -->
  <meta property="og:title" content="Your Page Title">
  <meta property="og:description" content="Description">
  <meta property="og:image" content="https://yoursite.com/img.jpg">
  <meta property="og:url" content="https://yoursite.com">

  <!-- Twitter Card -->
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:title" content="Your Page Title">

  <!-- Canonical URL (avoid duplicate content) -->
  <link rel="canonical" href="https://yoursite.com/page">

  <!-- Favicon -->
  <link rel="icon" href="favicon.ico" type="image/x-icon">

  <!-- Stylesheet -->
  <link rel="stylesheet" href="style.css">
</head>
Chapter 11 — Advanced

Data Attributes & HTML APIs

Custom data attributes let you store extra data directly in HTML elements for JavaScript to use.

Definition

Data attributes use the prefix data- followed by any name you choose. They store custom data without affecting the display. JavaScript can read them via element.dataset.

HTML + JS — data-* attributes
<!-- HTML: store data in elements -->
<button
  data-user-id="42"
  data-role="admin"
  data-city="Delhi"
  id="myBtn">
  Click Me
</button>

<script>
const btn = document.getElementById('myBtn');

// Read data attributes
console.log(btn.dataset.userId);   // "42"
console.log(btn.dataset.role);     // "admin"
console.log(btn.dataset.city);     // "Delhi"
</script>

HTML5 APIs You Should Know

HTML — Dialog, Details, Progress, Template
<!-- Native dialog box -->
<dialog id="modal">
  <p>I am a native modal!</p>
  <button onclick="modal.close()">Close</button>
</dialog>
<button onclick="modal.showModal()">Open Dialog</button>

<!-- Accordion / FAQ without JS -->

  <summary>Click to expand</summary>

<!-- Meter (like battery level) -->

Chapter 12 — Advanced

Accessibility (a11y)

Accessibility means making your website usable for everyone — including people using screen readers, keyboard-only navigation, or who have visual impairments.

HTML — Accessible Patterns
<!-- Always label form inputs -->
<label for="email">Email address:</label>
<input type="email" id="email">

<!-- ARIA: landmark roles -->
<div role="navigation">...</div>
<div role="main">...</div>
<div role="alert">Error: Invalid email</div>

<!-- ARIA: label elements with no visible text -->
<button aria-label="Close menu">✕</button>

<!-- ARIA: describe toggle state -->
<button aria-expanded="false">Open Menu</button>

<!-- Skip to content link (for keyboard users) -->
<a href="#main" class="skip-link">Skip to content</a>

<!-- Table with proper headers for screen readers -->
<table>
  <caption>Student Grades</caption>
  <tr>
    <th scope="col">Name</th>
    <th scope="col">Grade</th>
  </tr>
</table>

<!-- Focus management -->
<a href="#" tabindex="0">Focusable element</a>
<div tabindex="-1">Programmatically focusable only</div>
💡 ACCESSIBILITY CHECKLIST
✓ Every image has a descriptive alt text
✓ Every form input has a <label>
✓ Color is not the only way to convey information
✓ Page can be navigated using Tab key only
✓ Heading hierarchy is logical (h1 → h2 → h3)
